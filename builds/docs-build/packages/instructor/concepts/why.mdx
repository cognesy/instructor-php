---
title: 'Why use Instructor?'
description: 'Why a schema-first API is easier to work with.'
---

LLMs produce text well. Applications usually need typed data.

Instructor closes that gap by asking for a known shape and handling extraction, deserialization, validation, and retries before the result reaches your code.

## What You Gain

- Explicit response models
- Validation before you trust the data
- Retries for invalid outputs
- Streaming without changing the request shape
- A provider-agnostic API

```php
use Cognesy\Instructor\StructuredOutput;

final class User {
    public string $name;
    public int $age;
}

$user = (new StructuredOutput)
    ->with(messages: 'John is 34 years old.', responseModel: User::class)
    ->get();
// @doctest id="008c"
```
